# Security Policy

Rally is still early software.
Please report security problems carefully.

## Supported code

Security fixes land on the current `main` branch first.
Older branches and old commits may not get backports while Rally is still
pre-release.

## How to report a problem

- Do not open a public issue with exploit details.
- Use GitHub private vulnerability reporting for this repo if that option is
  available.
- If private reporting is not available, open a short public thread that only
  asks for a private contact path. Do not include the details there.

When you report a problem, include:

- what version, branch, or commit you tested
- the steps to reproduce it
- the impact you see
- any patch, mitigation, or proof-of-concept details you already have

## What to expect

The maintainer will try to confirm the report, understand the impact, and fix
the issue before sharing public details.
Because Rally is still pre-release, the fix may ship directly on `main`.
